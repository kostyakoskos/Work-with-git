Пусть кто-то другой изменил файл 1 в удаленном репоизитории. И мы тоже меняем этот файл 1. После git pull произойдет ошибка, т.к. 2 файла конфликтуют. Все проблемы нужно решать у
себя локально. На сервер отправлять рабочую версию.
Как решить? После git pull в файле 1 будут и мои изменения и изменения из сервера. Необходимо совместить вручную 2 изменения и отправить на сервер рабочую версию.
Потом делаем git add . git commit. git push.
